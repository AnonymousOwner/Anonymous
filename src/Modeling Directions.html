Modeling Tools

The Modeling Tools tab supports importing legal texts, and then modeling those same texts based on Massey's "A strategy for addressing ambiguity in regulatory requirements." Please see the "Help on Commands" link on the Modeling Tab for any user concerns.

End User Guide to functions in AHAB

Changing the Canvas Color

Default: Light Gray

In the Modeling Tools Tab: Find the canvas color drop down on the right of the screen (below the Tab values), select a value in the Canvas Color dropdown. The canvas should change to the color selected

Load HTML Files

Default: Not Applicable

In order to load an HTML file from the HTML files for import file in the directory, find the Load/Save Text Files panel. Select the button labeled Import, a pop-up box with the directory structure will appear. Select the HTML files for import folder, and then select one of the HTML files in the folder and click the Open button. The legal text will appear in the panel that defaults to LEGAL TEXT GOES HERE.

It is now possible to highlight the legal text and copy it into the Regulatory Text box (this is covered in the Capture Legal Text function section below).

Load/Save Ambiguity Models

Default: Not Applicable

Loading

In order to load a model from a JSON file, select the Import button from the Load/Save Model Files panel. A pop-up box will appear. Select the JSON folder from the file structure. Once the folder has opened, select one of the .json files by left clicking, and hitting the ‘Open’ button. The model file should then load in the canvas.

Saving/Exporting

In order to save a model to a JSON file, select the Export button from the Load/Save Model Files panel. A pop-up box will appear with a predefined format for the file. The format is as follows:

Model Name: Regulatory Text Field + Timestamp (Year, Month, Date – Time) + Perspective + version.json

The file will be saved into your browser’s predefined download folder. You will need to move the file into the JSON folder in your copy of the repository.

Model Palette Functions

Capture Legal Text

As discussed in the section Load HTML Files above, once a legal text is loading into the panel that is labeled LEGAL TEXT GOES HERE as a default, you may begin to highlight text that will be copied into the Regulatory Text field inside the Model Palette.

To copy text into the Regulatory Text Field hold the left click and highlight the text. Once done highlighting the text, click the Copy Text button and the data will be copied over into the Regulatory Text field to the right.

Modelling

Add an Ambiguity

The fields for the Ambiguity are:

  • Unique Artifact ID (should be auto-generated)
  • Regulatory Text ID (usually the HTML file designation e.g. USCCPASection2)
  • Perspective (values currently are: Eric, Aaron, Sepideh, Amin, Alden)
  • Regulatory Text (Copied by Copy Text button)
  • Regulatory Reference (values: Partial Phrase, Phrase, Sentence, Full Text)
  • Notes (free text field)
  • Ambiguity Type (values: Lexical, Syntactic, Semantic, Vagueness, Incompleteness, Referential, Other)
  • Severity (values: 1-5)
  • Intentionality (values: Yes or No)
  • Implementability (values: Yes or No)
NB: Link Weight is not needed for Ambiguities at this time.

Once all of the above fields contain values, select the Add button from the Modeling section. Once the function is selected (the field ‘Function Selected’ should show the word add), you can right click anywhere on the canvas to create the ambiguity. The Ambiguity values (see above) will clear except for the Unique Artifact ID, Regulatory Text ID, and perspective drop down.

Delete an Ambiguity

To delete an Ambiguity, left click on the item, and click the button labeled delete. The ambiguity and all links connected to the Ambiguity will be deleted. This cannot be undone! Multiple ambiguities can be deleted by left clicking and selecting the items for deletion.

IT IS RECOMMENDED YOU EXPORT MODEL FILES EARLY AND OFTEN!

Link an Ambiguity

To link an ambiguity with another element (currently ambiguities and terminals) select the first ambiguity by left clicking the ambiguity. Then left click another element. Now, press the link button. A link will be created, with a cardinality from the first element to the second element.

Creating Terminals

After creating ambiguities, you will want the start and stop terminals for the model. They are created by pressing the terminals button. Once created, you will want to link the terminals to the ambiguities via the link functionality. Terminals can be deleted and linked to, just like ambiguities.

Editing Ambiguities

To edit an ambiguity, select it by left clicking. Change the information inside the modeling panel. Once the information has changed, please click the edit button. The ambiguity will be updated to reflect the changes you have made.

Strategy

Heatmap Selection

The heatmap is the visual representation of the various Ambiguity elements (specifically: Implementability, Intentionality, Severity, and Ambiguity Type). To see these visual representations, simply select the value from the drop down. The canvas elements will change color and a small legend will appear to the left of the model.
